### Scenario 1: Voice-First User
- Uses voice for all answers
- Microphone button primary interaction
- Rarely clicks options
- Fast completion time

### Scenario 2: Click-First User
- Never uses voice
- Clicks all options
- Types in text fields

- Voice for open questions
- Clicks for multiple choice
- Types for notes
- Most realistic usage

## ğŸ’¡ Design Decisions Explained

**Why split screen?**
- Clear separation of concerns
- Voice panel doesn't distract from questions
- Both input methods always visible

**Why auto-play questions?**
- Reduces cognitive load
- Makes voice interaction natural
- Accessibility benefit

**Why large microphone button?**
- Primary interaction method
- Easy to target (especially on mobile)
- Visually prominent

**Why real-time transcript?**
- Build user confidence
- Catch errors immediately
- Transparency in AI interaction

**Why confetti?**
- Positive reinforcement
- Memorable ending
- Emotional connection

**Why coffee theme?**
- Relevant to domain
- Warm, inviting
- Professional yet friendly
